emacs 사용 도전#

emacs는 너무 배우기 어렵다. 라는 이야기를 많이 들어서 사용할 엄두를 못내다가 여러번의 도전 끝에 일단은 doom emacs이나 spacemacs 처럼 설정이 되어있는 emacs를 사용하자고 마음 먹었고, doom emacs로 emacs 라이프를 시작하였습니다.

격리된 config를 원하게 된 계기#

설정이 되어있는 doom emacs를 써보며, 내가 필요한 기능만 사용하면 좀 더 가볍게 사용할 수 있을 것이고, 설정을 하면서 배우는 점도 많을 것이라고 생각하게 되어 doom emacs의 설정을 유지하면서 깨끗한 상태의 emacs를 실행 할 수 있기를 바라게 되었습니다. 그래서 방법을 찾아보게 되었고, chemacs2를 이용해서 emacs 설정을 여러개 이용할 수 있다는 것을 알게 되었습니다.

$ [ -d ~/.emacs.d ] && mv ~/.emacs.d ~/.emacs.default
$ git clone https://github.com/plexus/chemacs2.git ~/.emacs.d

기존의 emacs 설정이 들어있는 .emacs 폴더의 이름을 .emacs.default로 변경하고 .emacs.d에 chemacs2를 클론합니다.

격리 방법#

  1. ~/.emacs-profiles.el 파일을 생성하고 내용을 입력합니다.

       (("default" . ((user-emacs-directory . "~/.emacs.default")))
   ("doom" . ((user-emacs-directory . "~/.emacs.doom.d"))))

  2. 이제 emacs를 실행할 때 –with-profile 플래그를 이용할 수 있고 실행시 원하는 configf를 고를 수 있습니다. 하지만 이 상태에서 실행을 하면 에러가 납니다.

       $ emacs --with-profile doom

이때 doom emacs의 설정 디렉터리의 .local/straight/repo 를 한번 지워주고 다시 빌드 합니다.

$ rm -rf ~/.emacs.doom.d/.local/straight/repo
$ doom build

이제 에러없이 제대로 실행 가능하게 되었습니다.#

# --with-profile 플래그를 사용하지 않으면 default값을 이용
$ emacs
$ emacs --with-profile doom

기억할 것!!!#

emacs를 건들기전에 설정파일을 무조건! 백업을 하고 진행해야, 지금까지 구축해놓은 설정을 다시 만들게 되는 불상사가 일어나지 않습니다!

이제부터는 설정도 하나하나 시간 날 때마다 해보려고 합니다. 그럼 오늘은 이만!
