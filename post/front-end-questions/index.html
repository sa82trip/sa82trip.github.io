화살표함수와 this 바인딩 ★★★#

refer this는 일반적으로 함수자신을 뜻하거나 class를 뜻함 arrow function은 this가 없다. 이 이야기는 this를 사용하면 외부에서 값을 가져온다.

let group = {
  title: "1모둠",
  students: ["보라", "호진", "지민"],

  showList() {
    this.students.forEach(
      student => alert(this.title + ': ' + student)
    );
  }
};

group.showList();

여기서 this.title은 group.title을 가리킨다

시맨틱 웹에 대한 이해 ★★★★#

refer HTML5에서는 시맨틱 웹을 쉽게 구성할수 있도록 만들어주는 요소들이 추가되었는데, 기존에

<div id="header"> 내용 </div>

로만 나타낼수 있던 머리 부분을

<header> 내용 </header>

로 쉽게 더 의미론적으로 나타낼수 있게 되었다.이는 div와 id=header 모두 읽었던 예전 XHTML이나 HTML4보다 훨씬 효율적이라고 볼수 있다.

  • 지금과 같이 사람만이 웹에 산재한 정보의 의미를 파악하는 것이 아닌, 자동화된 기계가 해석할 수 있는 일종의 표준 의미정보 교환의 수단이 되는 것이 시맨틱 웹의 목적이다. 시맨틱 웹의 이상향은, 인터넷에 방대한 양의 온톨로지가 산재하고, 이를 자동으로 해석하여 처리할 수 있는 에이전트 소프트웨어에 사람 또는 에이전트가 질의를 하면, 컴퓨터가 자동으로 분산된 온톨로지를 탐색하고 추론하여 원하는 결과를 돌려주는 것이다.

CommonJs, RequireJS ★#

  • CommonJS는 웹브라우저 밖의 JavaScript를 위한 모듈 생태계의 규칙을 설립하기 위한 프로젝트라고 합니다.

  • 웹브라우저 밖의 JavaScript라 하면 대표적으로 Nodejs

  • Nodejs는 기본적으로 CommonJS의 규칙을 따릅니다.

      // a.js
  const printHelloWorld = () => {
    console.log('Hello Wolrd');
  };

  module.exports = {
    printHelloWorld
  };

//b.js
const func = require('./a.js'); // 같은 디렉토리에 있다고 가정
func.printHelloWorld();
  • CommonJS의 모듈 제공 방식은 두가지가 있습니다.
    • module.exports & exports
  • AMD(Asynchronous Module Definition)
    • RequireJS => 대표적 모듈 시스템
      • 클라이언트 사이드에서 모듈을 로딩할 때, 스크립트 태그를 이용해서 로딩하는 것은, 로딩이 완료될 때까지 브라우저가 프리징되어 사용자 경험에 안좋은 영향을 끼쳤습니다. 그래서 비동기적으로 모듈 로딩을 할 필요가 있었기 때문에 태어났습니다. 하지만! 이전에 태어난 배경이 그랬던 것이지, 최근에는 대안이 존재합니다.

ESM(ECMAScript Module)#

  • ESM은 ES6부터 지원되는 자바스크립트 자체 모듈 시스템입니다.
  • HTML 파일에서 직접 가져오는 코드는 아래와 같이 작성합니다.
<html>
  //...
  <script type="module" src="lib.mjs"></script>
  <script type="module" src="app.mjs"></script>
  //...
</html>
  • ES6모듈 파일의 확장자는 모듈임을 명확하게 하기 위해 mjs를 사용한다고 하네요!
  • 그럼 js 파일 내에서 사용하는 방법을 알아볼까요? export

모듈밖으로 내보낼 때는 export키워드를 이용합니다.

// module.mjs
export const pi = Math.PI;

export function square(x){
  return x * x;
}

export class Person {
  constructor(name) {
    this.name = name;
  }
}

차례로 변수, 함수, 클래스를 내보내는 방법입니다. 간단하죠? 또는 한번에 모아서 내보내는 방법도 있습니다.

// module.mjs
const pi = Math.PI;

function square(x) {
  return x * x;
}

class Person {
  constructor(name) {
    this.name = name;
  }
}

export { pi, square, Person };

import#

모듈을 가져올 때는 import키워드를 사용합니다. export된 식별자를 그대로 가져오며 확장자는 생략 가능합니다(mjs).


// app.mjs
import { pi, square, Person } from './module.mjs';

console.log(pi); // 3.14...
console.log(square(10)); // 100
console.log(new Person('Lee')); Person { name: 'Lee' }

어떤가요? 제가 느끼기엔 다른 모듈 시스템보다 훨씬 직관적이고 깔끔해보이네요👍

한번에 내보내는 방법이 있었으니 한번에 가져오는 방법도 있겠죠?

// app.js
import * as lib from './module.mjs';

console.log(lib.pi);
console.log(lib.square(10));
console.log(new lib.Person('Lee'));
이렇게하면 export된 데이터들을 한번에 가져올 수 있습니다!

또, 가져오면서 자신이 원하는 이름으로 변경해서 가져올 수도 있습니다.

// app.js
import { pi as PI, square as sq, Person as P } from './module.mjs';

console.log(PI);
console.log(sq(2));
console.og(new P('Lee'));

export default#

  • 하나의 데이터만을 export할 때는 default 키워드를 사용합니다.
// module.mjs
export default function (x) {
  return x * x;
}

deafult키워드를 사용하는 경우에는 var, let, const를 사용할 수 없습니다.

// module.mjs
export default () => {}; // OK

export default const foo = () => {}; // Error

default키워드로 export된 모듈은 {} 없이 임의의 이름으로 import합니다.

// module.mjs
import square from './module.mjs';

console.log(square(3)); // 9

Package-lock.json이란? ★#

package.json? package-lock.json?#

  • package.json is a file that contains information about your project (name, version, etc) and it lists the packages that your project is dependent on.

package.json#

  • 노드로 작성하는 애플리케이션도 package.json 파일을 사용하여 관리
  • package.json 파일을 이용하면 사용하는 확장 모듈에 대한 의존성 관리가 가능하기 때문에 편리
  • pacakge.json 파일은 기본적으로 CommonJS의 명세를 충실히 따르고 있으며 JSON 형식의 파일
  • dev 할 때만 사용하는 dependency들도 추가 가능

problem#

“^” mark 의 의미는 ^2.20.0 버젼의 페키지가 현재 내 프로젝트에서 쓰이고 있는데, 나중에 누군가 프로젝트를 클론하여 npm install 할 때 마이너 버전이 달라질 수 있다는 이야기

package-lock.json#

package-lock.json will simply avoid this general behavior of installing updated minor version so when someone clones your repo and run npm install in their machine. NPM will look into package-lock.json and install exact versions of the package as the owner has installed so it will ignore the ^ and ~ from package.json.

XSS, CSRF의 개념과 방어방법 ★★★★#

XSS#

XSS(Cross Site Scripting) 웹 브라우저에서 사용자가 입력 할수 있는 input태그 등에 악의적인 script를 작성하여 해당 contents를 이용하는 다른 이용자의 개인정보 및 쿠키정보 탈취, 악성코드 감염, 웹 페이지 변조등의 공격을 한다.

  • 사용자에게 입력 받은 값을 철저히 검증하는 것이 중요

  • script를 사용할 수 없게 막는 것이 중요 “<” ,">" 같은 문자들은 < >와 같은 html 코드를 이용해서 표현한다.

      function XSSFilter(content) {
    return content.replace(/</g, "&lt;").replace(/>/g, "&gt;");
  }

  • XSS 공격에 대한 방어 라이브러리를 사용한다

  • 반사적, 지속적, dom

CSRF#

정의#

  • CSRF 공격(Cross Site Request Forgery)은 웹 어플리케이션 취약점 중 하나로 인터넷 사용자(희생자)가 자신의 의지와는 무관하게 공격자가 의도한 행위(수정, 삭제, 등록 등)를 특정 웹사이트에 요청하게 만드는 공격입니다.

  • XSS가 사용자가 특정 사이트를 신뢰한다는 점을 공격하는거라면, CSRF는 특정 사이트가 사용자의 브라우저를 신뢰한다는 점을 공격하는 것이 다르다.

방어방법#

  1. Referrer 검증

request header에 있는 요청을 한 페이지의 정보가 담긴 referrer 속성을 검증하여 차단. 일반적으로 이 방법만으로도 대부분 방어가 가능할 수 있다. 옥션이 아닌 개인 이메일에서 요청이 들어오는 것처럼, 같은 도메인 상에서 요청이 들어오지 않는다면 차단하도록 하는 것이다.

  1. CSRF Token 사용

랜덤한 수를 사용자의 세션에 저장하여 사용자의 모든 요청(Request)에 대하여 서버단에서 검증하는 방법.

// 로그인시, 또는 작업화면 요청시 CSRF 토큰을 생성하여 세션에 저장한다. session.setAttribute(“CSRF_TOKEN”,UUID.randomUUID().toString());

// 요청 페이지에 CSRF 토큰을 셋팅하여 전송한다

  1. CAPTCHA 사용

요즘은 거의 모든 웹사이트에서 캡차를 이용하는 것 같은데 캡차이미지상의 숫자/문자가 아니라면 해당 요청을 거부하는 것이다.

이 외에도 form 태그를 입력할 시 GET방식을 지양하고 POST방식을 쓰도록 하는 것은 기본이라고 할 수 있다.

Session#

  • 정의
    1. 망 환경에서 사용자 간 또는 컴퓨터 간의 대화를 위한 논리적 연결.
    2. 프로세스들 사이에서 통신을 하기 위해 메시지 교환을 통해 서로를 인식한 이후부터 통신을 마칠 때까지의 기간.
  • cookie와의 관계
    • 쿠키는 서버가 session을 구현하기 위해 쓰인다.
    • 쿠키는 세션의 identifier를 포함한다
  • web framework의 session과 cookie를 다루는 기능은 중요한 파트 중에 하나
    • 쿠키가 존재한다면 쿠키를 이용해서 session data를 찾는다
    • 쿠키가 없다면 새로운 session과 cookie를 만든다
  • session을 다루는 접근법들
    1. Approach 1: just keep in main memory
    2. Approach 2: store session state in file on disk
    3. Approach 3: store session state in a database
  • 서버는 결국 오래된 session data를 지운다
  • session은 몇몇의 보안 이슈가 있다.

Cookie#

  • 정의
    • 특정 웹사이트에 방문시 만들어지는 정보를 담고있는 파일
      • 방문자의 pc에 저장
      • 용량은 주로 4kb 미만
      • 여러 쿠키가 생성될 수는 있지만 브라우져는 50개 정도로 제한한다.

HTTP 메서드가 하는 역할#

request#

  • GET
    • GET 메서드는 특정 리소스의 표시를 요청합니다. GET을 사용하는 요청은 오직 데이터를 받기만 합니다.
  • HEAD
    • HEAD 메서드는 GET 메서드의 요청과 동일한 응답을 요구하지만, 응답 본문을 포함하지 않습니다.
  • POST
    • POST 메서드는 특정 리소스에 엔티티를 제출할 때 쓰입니다. 이는 종종 서버의 상태의 변화나 부작용을 일으킵니다.
  • PUT
    • PUT 메서드는 목적 리소스 모든 현재 표시를 요청 payload로 바꿉니다.
  • DELETE
    • DELETE 메서드는 특정 리소스를 삭제합니다.
  • CONNECT
    • CONNECT 메서드는 목적 리소스로 식별되는 서버로의 터널을 맺습니다.
  • OPTIONS
    • OPTIONS 메서드는 목적 리소스의 통신을 설정하는 데 쓰입니다.
  • TRACE (en-US)
    • TRACE 메서드는 목적 리소스의 경로를 따라 메시지 loop-back 테스트를 합니다.
  • PATCH
    • PATCH 메서드는 리소스의 부분만을 수정하는 데 쓰입니다.

payload#

등록된 (registered) 클레임, 공개 (public) 클레임, 비공개 (private) 클레임

  • registered #1 등록된 (registered) 클레임 등록된 클레임들은 서비스에서 필요한 정보들이 아닌, 토큰에 대한 정보들을 담기위하여 이름이 이미 정해진 클레임들입니다. 등록된 클레임의 사용은 모두 선택적 (optional)이며, 이에 포함된 클레임 이름들은 다음과 같습니다:

    iss: 토큰 발급자 (issuer) sub: 토큰 제목 (subject) aud: 토큰 대상자 (audience) exp: 토큰의 만료시간 (expiraton), 시간은 NumericDate 형식으로 되어있어야 하며 (예: 1480849147370) 언제나 현재 시간보다 이후로 설정되어있어야합니다. nbf: Not Before 를 의미하며, 토큰의 활성 날짜와 비슷한 개념입니다. 여기에도 NumericDate 형식으로 날짜를 지정하며, 이 날짜가 지나기 전까지는 토큰이 처리되지 않습니다. iat: 토큰이 발급된 시간 (issued at), 이 값을 사용하여 토큰의 age 가 얼마나 되었는지 판단 할 수 있습니다. jti: JWT의 고유 식별자로서, 주로 중복적인 처리를 방지하기 위하여 사용됩니다. 일회용 토큰에 사용하면 유용합니다.

  • public 공개 클레임들은 충돌이 방지된 (collision-resistant) 이름을 가지고 있어야 합니다. 충돌을 방지하기 위해서는, 클레임 이름을 URI 형식으로 짓습니다.

      {
      "https://velopert.com/jwt_claims/is_admin": true
  }

  • private #3 비공개 (private) 클레임 등록된 클레임도아니고, 공개된 클레임들도 아닙니다. 양 측간에 (보통 클라이언트 <->서버) 협의하에 사용되는 클레임 이름들입니다. 공개 클레임과는 달리 이름이 중복되어 충돌이 될 수 있으니 사용할때에 유의해야합니다.

      {
      "username": "velopert"
  }

signature#

JSON Web Token 의 마지막 부분은 바로 서명(signature) 입니다. 이 서명은 헤더의 인코딩값과, 정보의 인코딩값을 합친후 주어진 비밀키로 해쉬를 하여 생성합니다.

서명 부분을 만드는 슈도코드(pseudocode)의 구조는 다음과 같습니다.

HMACSHA256( base64UrlEncode(header) + “.” + base64UrlEncode(payload), secret) 이렇게 만든 해쉬를, base64 형태로 나타내면 됩니다 (문자열을 인코딩 하는게 아닌 hex → base64 인코딩을 해야합니다)
