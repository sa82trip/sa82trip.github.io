변수의 스코프 차이 (let과 var의 차이)#

미리 알아야 하는 사항! javascript는 block scope를 가지는 게 아니라

  • function
  • with
  • catch

이렇게 세군대에서 scope가 생성이 됩니다

아래 코드는 es6에서 가능하게 된 코드입니다

//es6
let callbacks = [];
for(let i=0 ; i<=2 ; i++){
    callbacks[i] = function () {return i * 2}
}

console.log(callbacks[0]() === 0)
console.log(callbacks[1]() === 2)
console.log(callbacks[2]() === 4)

#+RESULTS:
: true
: true
: true
: undefined

여기서 0,2,4로 각기 다른 i 값을 가지고 2를 곱하여 값을 리턴하고 있는데, 이게 es6이전에는 이런 식으로 block scope를 생성할 수 없었습니다.

아래의 코드를 실행하고 div들을 클릭하여보면 뭔가 문제가 있다는 걸 알수있는데, 그건 바로 scope에 관한 문제인데, scope가 언제 만들어지는지, 언제까지 유지가 되는지를 알면 이해하기 쉽습니다.

<div id="div0">Click me! DIV 0</div>
<div id="div1">Click me! DIV 1</div>
<div id="div2">Click me! DIV 2</div>
<script>
   var i, len = 3; for (i = 0; i < len; i++) {                              //#1
       document.getElementById("div" + i).addEventListener("click",
           function () {                                                    //#2
               alert("You clicked div #" + i);                              //#3
       }, false); }
</script>

위의 코드로 html파일을 만들고 열어서 div를 클릭해보면 “You clicked div 3"만 나올 것입니다. 그 이유는 scope가 만들어지지 않고, global scope의 i를 가리키고 있고, 모든 div의 event 함수가 루프가 다 돌고 난 후의 i 값인 3을 가리키고 있기 때문입니다.

  • 7 line은 global scope i를 참조하는것을 방지하기 위해서 익명함수를 만들어 익명함수의 인자값을 리턴하는 함수를 리스트에 넣으면서 i값을 인자값으로 주었고, 결과적으로 루프가 돌아가면서 i값을 index로 받는 순간 스코프가 만들어져서 모두 각자의 스코프가 만들어졌습니다.
  • 13 line은 루프를 돌면서 j값을 return 하는 익명함수를 list2에 넣었는데, 함수를 만드는 순간에 참조하게 되는 스코프가 global scope의 j이고, 루프가 돌면서 3으로 바뀐 j의 값을 list2의 0번, 1번, 2번방의 함수들이 참조하고 있어서 console.log를 찍었을 때 모두 3이 나오게 됩니다.

이제 다시 원래 주제로 돌아와서, es6에서 바뀐점이 let, const를 사용할 수 있다는 것인데요, let을 이용하면 block scope를 사용할 수 있다는 것을 말하고 싶었던 것인데 저도 좀 더 자세히 이해하고 싶어서 길게 늘여서 쓰게 된 것같습니다!

마무리 정리#

  • es6에서는 const와 let 키워드를 사용할 수 있게 되었습니다.
  • const는 immutable을 위해서 사용하고 (기억해야할 점은 primitive타입이 아닌경우는 imuutable이 아니라는 점입니다…)
  • let은 block scope를 생성하기 위해 사용됩니다.

  • javascript core

    • scope를 생성하는 3가지
      • function
      • with
      • catch

  • 혹시 잘못된 점을 발견하시고 리플로 가르침을 주신다면 정말 감사하겠습니다.
