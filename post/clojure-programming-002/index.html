naming with “def”#

def를 이용해서 vector에 이름 정하기 clojure에서는 name에 vector를 binding 한다고 표현합니다.

(def my-favorite-devices
  ["magic keyboard", "Magic Trackpad", "AOC monitor"])

my-favorite-devices

#+RESULTS:
| #'user/my-favorite-devices                        |
| ["magic keyboard" "Magic Trackpad" "AOC monitor"] |

다른 언어에서는 assign이라고 표현하고 variable에 몇번의 re-assign이 가능한 언어들도 있지만 clojure는 그렇지 않습니다.

(def severity :mild)
(def error-message "This is Soooooo~")
(if (= severity :mild)
  (def error-message (str error-message "Hot!!!!!!"))
  (def error-message (str error-message "Sweet!!!!!")))

#+RESULTS:
| #'user/severity      |
| #'user/error-message |
| #'user/error-message |

error-message에 두번의 binding을 하려고 했기 때문에 에러가 났습니다.

이럴 때는 다른 방법들도 나중에 사용해보겠지만 우선 함수를 이용해서 하는 방법입니다.

(defn error-message
  [severity]
  (str "This is Soooooo ~ "
       (if (= severity :mild)
         "Hot!!!!!"
         "Sweet!!!!!")))
(error-message :mild)

#+RESULTS:
| #'user/error-message         |
| "This is Soooooo ~ Hot!!!!!" |

함수를 만드려면 defn을 사용하면 됩니다. 추후에 함수에 대해서 좀 더 정확히 공부할 것이고, 일단은 def는 constants를 정의한다고 생각하면 될 것 같습니다.

Data Structures#

모든 clojure의 data structure는 immutable입니다. 다른 언어를 예를 들면 javascript로 예를 들어보겠습니다.

const aList = [1, 2, 3];
console.log(aList);
aList[0] = 100;
console.log(aList);

#+RESULTS:
: [ 1, 2, 3 ]
: [ 100, 2, 3 ]
: undefined

이렇게 0번째 방의 값을 100으로 바꿀 수가 있는데요, clojure에서는 이러한 행동을 할 수 없습니다.

Number#

100
3.4
1/3

#+RESULTS:

"고구려는 멋진 나라였다."

Maps#

clojure는 hash map과 sorted map이 있다.

{:name "Gildong Go"
 :gender "Male"}

#+RESULTS:
: {:name "Gildong Go", :gender "Male"}
:

{"test-name" -}

#+RESULTS:
: {"test-name" #function[clojure.core/-]}
:

{:address {:country "korea" :city "seoul"}}

#+RESULTS:
: {:address {:country "korea", :city "seoul"}}
:

hash-map함수를 이용해서 map만들기

(hash-map :name "Gildong" :gender "Mail")

#+RESULTS:
: {:name "Gildong", :gender "Mail"}
:

값 찾기

(get {:name "Gildong" :gender "Male"} :gender)

#+RESULTS:
: Male

값 찾으면서 default value 지정하기

(get {:name "Gildong" :gender "Male"} :age "50?")

#+RESULTS:
: 50?

nested map에서 값 찾기 with get-in

(get-in {:address {:country "korea" :city "seoul"}} [:address :city])

#+RESULTS:
: "seoul"
:

map을 인자값을 받는 함수처럼 이용해서 값 찾기

({:name {:first "Gildong" :last "Go"} :gender "Male"} :name)

#+RESULTS:
: {:first "Gildong", :last "Go"}
:

마지막으로 keyword를 함수로 이용해서 값을 찾을 수 있습니다.

(:name {:name {:first "Gildong" :last "Go"} :gender "Male"})

#+RESULTS:
: {:first "Gildong", :last "Go"}
:

Keywords#

:name
:112
:_?

#+RESULTS:
| :name                                         |
| :112                                          |
| :_                                            |
| class clojure.lang.Compiler$CompilerException |

“:_?” org-babel에서 예외발생하는데, 아직 해결 못했습니다.

;;두줄의 코드는 같은 의미 입니다.
(:name {:name "Gildong" :age 45})
(get {:name "Gildong" :age 45} :name)

keyword를 함수를 사용하는 방법도 default value를 이용할 수 있습니다.

(:personality {:name "Gildong" :age 45} "very kind and warm")
