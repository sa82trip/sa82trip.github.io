aws lambda#

비정기 공부회에서 HKS상이 공유한 내용 정리#

  • Faas (Function As A Service)이다.
    • Saas의 다음 단계
  • 장점:
    • 쓴만큼 낸다 백만콜에 20센트정도
    • auto scailing
    • 높은 가용성
  • 단점:
    • cold boot가 있다.
    • 속도가 ec2보다 약간 느릴 수 있다.
    • Migrating is not that simple and easy

간단 구성법#

  1. aws console page 검색창에서 lamda를 검색한다

  2. 함수생성 버튼을 누르고 생성을 시작한다

  3. 함수이름, 런타임(랭귀지등)을 선택한다.(보안관련 설정은 나중에 좀 더 공부해야함.)

  4. 완성되면 구성탭에 들어가서 endpoint를 보면 접근할 수 있는 주소가 나온다.

  5. 접근을 하려면 trigger를 설정해야한다

  6. 기본적으로 http요청에 대한 설정을 하려면 api 게이트웨이를 트리거에 설정해준다

  7. python기준으로 lamda_function.py파일에 lamda_handler함수가 유일하게 실행되는 함수라는 것을 기억한다

  8. 코드를 올리는 방법 3가지

    1. 직접작성 (코드가 5mb이상이면 안되기 때문에 아래의 방법들을 사용한다)
    2. zip파일로 올리기
    3. s3위치를 제공하기
      1. s3 bucket을 만든다(unique한 이름)
      2. s3로 시작하는 주소를 이용해서 lamda에 적용한다

  9. 구성이 완료된 모습

  10. 환경변수 설정하는 법

  11. lambda_handler

        # python
    import sys
    sys.path.append('./libs')
    from bs4 import BeautifulSoup
    import json
    import os
    import requests

    NAME = os.environ['name']

    # Main
    def lambda_handler(event, context):
        authenticated = None
        try:
            if 'queryStringParameters' in event:
                if 'authenticated' in event['queryStringParameters']:
                    result_list = []
                    bank_code = event['queryStringParameters']['authenticated']
                    url = "http://www.cgv.co.kr/movies/"
                    res = requests.get(url=url)
                    bs4_obj = BeautifulSoup(res.content, 'html.parser')
                    titles = bs4_obj.select(".title")
                    for title in titles:
                        result_list.append(title.text)
                    print(result_list)
            else:
                raise TypeError()

            return {
                'statusCode': 200,
                'headers': {"content-type": "application/json; charset=utf-8"},
                'body': json.dumps(result_list)
            }

        except TypeError:
            return {
                'statusCode': 400,
                'headers': {"content-type": "application/json; charset=utf-8"},
                'body': json.dumps('Need More Parameters.')
            }
