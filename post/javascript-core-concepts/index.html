1. call by reference or value?#

primitive type use value for copying#

let var1 = "var1"
let var2 = var1
console.log(var2);
var1 = "modified";
console.log(var2);

#+RESULTS:
: var1
: var1

var1의 값을 바꿨지만 var2의 값이 바뀌지 않았다는 것은 var1을 참조하고 있지 않다는 의미입니다.

Object, Array, Function use reference for copying#

  • Object

      let var1 = {name:"firebat"}
  let var2 = var1
  console.log(var2)
  var2.name="ali"
  console.log("var2",var2)
  console.log("var1",var1)

        #+RESULTS:
      : { name: 'firebat' }
      : var2 { name: 'ali' }
      : var1 { name: 'ali' }
      : undefined

    var2의 값을 바꿨는데 var1도 바뀌었다는 것은 var2가 var1의 객체의 주소를 참조하고 있기 때문에 같은 객체를 바라보고 있는 중이라는 의미입니다.

  • Array

      let var1 = [1,2,3,4,5];
  let var2  = var1;
  console.log(var2)
  var2[0]=100
  console.log("var2",var2)
  console.log("var1",var1)

        #+RESULTS:
      : [ 1, 2, 3, 4, 5 ]
      : var2 [ 100, 2, 3, 4, 5 ]
      : var1 [ 100, 2, 3, 4, 5 ]
      : undefined

    var2의 값을 바꿨는데 var1도 바뀌었다는 것은 var2가 var1의 객체의 주소를 참조하고 있기 때문에 같은 객체를 바라보고 있는 중이라는 의미입니다.

2. Closure#

Clojure가 아닙니다. ☺️ 종료된 함수의 scope에 접근이 가능하게 해줍니다.

function createGreeter(greeting) {
    return function(name) {
    console.log(greeting + ', ' +  name)
    }
}
const sayHello = createGreeter("hello~!");
console.log(sayHello);
sayHello('Gil');

#+RESULTS:
: [Function (anonymous)]
: hello~!, Gil

3. Destructuring (구조파괴)#

js에서 object, array등에서 원하는 부분을 추출할 때 자주 사용됩니다.

const interviewer1 = {
    name: 'Peterson',
    age: 29,
    military: "finished duty",
        }

const {name, age, military} = interviewer1;

console.log(`${name} is ${age} years old and he ${military} military service`);

#+RESULTS:
: Peterson is 29 years old and he finished duty military service
: undefined
